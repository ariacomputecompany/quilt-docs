Quilt is a runtime and control plane for containers, storage, orchestration, clusters, serverless functions, and Kubernetes-style workflows.

These docs are designed for two kinds of readers at the same time:

- someone touching Quilt for the first time
- someone who already knows the product and needs exact API behavior fast

<Tip>
  If you are new, start with <a href="/quickstart">Quickstart</a>, then read <a href="/concepts/mental-model">Mental model</a>. That path gives you confidence before you dive into the larger API surface.
</Tip>

## The Quilt way to troubleshoot

When something feels confusing, do this in order:

<Steps>
  <Step title="Check health">
    Start with `GET /health`, then use the concern-specific health route like `/api/containers/health`.
  </Step>
  <Step title="Find the exact target">
    Resolve the container, function, workload, or cluster you actually mean.
  </Step>
  <Step title="Check readiness, not just existence">
    Created is not the same thing as ready.
  </Step>
  <Step title="Inspect the returned handle">
    Operations and exec jobs are part of the product contract. Follow them.
  </Step>
</Steps>
